# Release procedure

Copyright (c) 2026 Pedro Martins (pedro5g)

Rebyte releases are generated by `dist` 0.32.0 from
`dist-workspace.toml`. Only `rebyte-cli` is distributed; libraries and signing
adapters remain workspace source packages rather than separate binaries.

## Before tagging

1. Confirm the worktree is clean and `main` contains the intended commits.
2. Update the workspace version and move `CHANGELOG.md` entries from
   `Unreleased` into that version and date.
3. Run `cargo xtask check`, `cargo xtask test`, the Wasm target check and the
   fuzz-harness compile check.
4. Run `cargo xtask security` with current `cargo-audit` and `cargo-deny`.
5. Run `dist generate --check` and `dist plan`; inspect every target and asset.
6. Confirm no production secret, token, `.ai/` content or `.rebyte/` journal is
   tracked.

## Publish

Create and push a signed `vMAJOR.MINOR.PATCH` tag. The generated release
workflow builds six target archives, shell/PowerShell installers, SHA-256
checksums, a source tarball and a CycloneDX SBOM. Builds include auditable Cargo
metadata and GitHub artifact attestations.

No automation may silently replace a tag or existing release asset.

## Verify the release

1. Download every archive and its `.sha256` file and verify the checksum.
2. Verify GitHub provenance with `gh attestation verify --repo pedro5g/Rebyte`.
3. Inspect the CycloneDX SBOM and run a RustSec audit of embedded metadata.
4. Run `rebyte doctor`, `inspect`, `verify` and a dry-run apply on each supported
   operating system.
5. Run the documented key-generation, signed `pack`, external-key `verify` and
   RAP `hash --check` smoke tests with disposable release fixtures.
6. Confirm installer and archive binaries report the tagged version.
7. Record any release exception in `CHANGELOG.md`; never edit an RAP v1 test
   vector after publication.

The official `dist` configuration reference and supply-chain guide are at
<https://axodotdev.github.io/cargo-dist/book/reference/config.html> and
<https://axodotdev.github.io/cargo-dist/book/supplychain-security/index.html>.
